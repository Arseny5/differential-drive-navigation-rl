# Отчёт: Навигация робота с дифференциальным приводом методами обучения с подкреплением

## 1. Описание среды

### 1.1 Задача

Робот с дифференциальным приводом должен проехать от случайной стартовой точки (квадрант III: x in [-2, 0], y in [-10, 0]) к случайной цели (квадрант I: x in [0, 2], y in [0, 10]), обходя круговое препятствие радиуса 2, расположенное в начале координат.

### 1.2 Кинематическая модель

Состояние робота: (x, y, theta) — позиция и угол ориентации. Управление: скорости левого и правого колёс (v_l, v_r) in [-2, 2].

Уравнения движения (дискретизация Эйлера, dt = 0.1):

    x_{t+1} = x_t + (v_r + v_l)/2 * cos(theta_t) * dt
    y_{t+1} = y_t + (v_r + v_l)/2 * sin(theta_t) * dt
    theta_{t+1} = theta_t + (v_r - v_l) / L * dt

где L = 0.5 — расстояние между колёсами (wheelbase).

### 1.3 Пространство наблюдений (8D)

| Компонента | Описание |
|-----------|----------|
| x, y | Позиция робота |
| cos(theta), sin(theta) | Ориентация (без разрывов) |
| g_x, g_y | Координаты цели |
| d_goal | Евклидово расстояние до цели |
| d_obs | Евклидово расстояние до центра препятствия |

Кодирование угла через cos/sin вместо theta напрямую исключает разрыв на границе [-pi, pi] и улучшает стабильность обучения.

### 1.4 Пространство действий

Непрерывное, 2D: [v_left, v_right] in [-2.0, 2.0]. Действия клиппируются после выхода из политики.

### 1.5 Функция награды

Формирующая (shaped) награда, сочетающая три компоненты на каждом шаге:

    r_t = -0.05 (step penalty) + 5.0 * (d_{t-1} - d_t) (progress) - 0.01 * |v_r - v_l| (spin penalty)

Терминальные бонусы:
- Достижение цели (d_goal < 0.5): +100
- Столкновение с препятствием (d_obs < 2.0): -100
- Превышение 500 шагов: усечение (truncation), без бонуса

**Обоснование:**
- Штраф шага (-0.05) стимулирует быстрое достижение цели.
- Прогресс-компонента (масштаб 5.0) задаёт плотный обучающий сигнал — агент получает положительную награду при приближении к цели на каждом шаге.
- Штраф вращения предотвращает "кружение на месте" и поощряет плавное движение.
- Крупные терминальные бонусы (+100 / -100) чётко отделяют успех от столкновения.

### 1.6 Условия завершения

1. Достижение цели: расстояние до цели < goal_radius (0.5) — terminated, success.
2. Столкновение: расстояние до центра < obstacle_radius (2.0) — terminated, failure.
3. Тайм-аут: step_count >= 500 — truncated.

## 2. Реализованные алгоритмы

Все алгоритмы используют одинаковую архитектуру нейросетей:
- **Policy Network:** MLP [128, 128] с активацией Tanh, выход — среднее Гауссова распределения, обучаемый log_std (зажат в [-2.0, 0.5]).
- **Value Network** (для методов с baseline/critic): MLP [128, 128] с Tanh, выход — скаляр V(s).

### 2.1 REINFORCE (Vanilla Monte Carlo Policy Gradient)

**Идея:** Оценка градиента политики по полным возвратам эпизода.

**Формула обновления:**

    nabla_theta J = E[ sum_t nabla_theta log pi_theta(a_t|s_t) * G_t ]

    G_t = sum_{k=0}^{T-t} gamma^k * r_{t+k}

**Особенности реализации:**
- Нормализация возвратов (mean/std) для стабильности.
- Клиппирование нормализованных возвратов в [-5, 5].
- Энтропийная регуляризация (coef = 0.05) для поддержания исследования.
- Обновление после каждого эпизода.
- Gradient clipping (max_norm = 0.5).

**Гиперпараметры:**
- Learning rate: 3e-4
- Gamma: 0.99
- Entropy coef: 0.05

**Достоинства:** Простота, отсутствие смещения (unbiased).
**Недостатки:** Высокая дисперсия градиента, медленная сходимость.

### 2.2 REINFORCE + Baseline

**Идея:** Добавление обученной функции ценности V_phi(s) в качестве baseline для снижения дисперсии.

**Формула обновления:**

    nabla_theta J = E[ sum_t nabla_theta log pi_theta(a_t|s_t) * A_t ]

    A_t = G_t - V_phi(s_t)

Baseline V_phi обучается минимизацией MSE: L_V = E[(V_phi(s_t) - G_t)^2].

**Особенности реализации:**
- Отдельные оптимизаторы для policy и value.
- Нормализация advantages.
- Entropy coef: 0.02.

**Гиперпараметры:**
- Policy LR: 3e-4, Value LR: 1e-3
- Gamma: 0.99

**Достоинства:** Меньшая дисперсия по сравнению с ванильным REINFORCE.
**Недостатки:** Всё ещё Monte-Carlo возвраты (полный эпизод).

### 2.3 Actor-Critic (A2C + GAE)

**Идея:** Замена Monte-Carlo возвратов на TD-оценки с Generalized Advantage Estimation.

**GAE:**

    A_t^GAE = sum_{l=0}^{inf} (gamma * lambda)^l * delta_{t+l}

    delta_t = r_t + gamma * V(s_{t+1}) - V(s_t)

**Особенности реализации:**
- Буфер на rollout_episodes = 8 эпизодов перед обновлением — повышает стабильность.
- GAE lambda = 0.95.
- Нормализация advantages по всему rollout.
- Entropy coef: 0.02.

**Гиперпараметры:**
- Policy LR: 3e-4, Value LR: 1e-3
- Gamma: 0.99, GAE lambda: 0.95

**Достоинства:** Баланс смещения и дисперсии, более частые обновления.
**Недостатки:** Смещение из-за бутстрэпинга.

### 2.4 TRPO (Trust Region Policy Optimization)

**Идея:** Оптимизация суррогатной функции с ограничением на KL-дивергенцию между старой и новой политиками. Натуральный градиент вычисляется через сопряжённые градиенты (Conjugate Gradient).

**Задача оптимизации:**

    max_theta E[ pi_theta(a|s) / pi_theta_old(a|s) * A(s,a) ]
    s.t. D_KL(theta_old || theta) <= delta

**Алгоритм:**
1. Собрать rollout (8 эпизодов).
2. Вычислить GAE advantages.
3. Получить градиент суррогатной функции g.
4. Решить Fx = g методом сопряжённых градиентов (10 итераций), где F — матрица Фишера.
5. Вычислить шаг: step = sqrt(2*delta / x^T F x) * x.
6. Линейный поиск с коэффициентом отката 0.5 (до 10 итераций).

**Гиперпараметры:**
- Value LR: 1e-3
- Max KL: 0.01, CG iters: 10, Damping: 0.1
- GAE lambda: 0.95

**Достоинства:** Гарантированное монотонное улучшение, стабильность.
**Недостатки:** Вычислительно дороже из-за CG и line search.

### 2.5 PPO (Proximal Policy Optimization)

**Идея:** Аппроксимация trust region через клиппирование отношения вероятностей. Проще TRPO, но сохраняет стабильность обновлений.

**Функция потерь:**

    L^CLIP = E[ min( r_t(theta) * A_t, clip(r_t(theta), 1-eps, 1+eps) * A_t ) ]

    r_t(theta) = pi_theta(a_t|s_t) / pi_theta_old(a_t|s_t)

**Особенности реализации:**
- Буфер на rollout_episodes = 8 эпизодов.
- Несколько эпох обновления (ppo_epochs = 4) по мини-батчам (size = 64).
- Клиппирование epsilon = 0.2.
- Отдельные оптимизаторы для policy и value.

**Гиперпараметры:**
- Policy LR: 3e-4, Value LR: 1e-3
- Clip epsilon: 0.2, PPO epochs: 4, Mini-batch: 64
- GAE lambda: 0.95

**Достоинства:** Простота реализации, хорошая производительность, стабильность.
**Недостатки:** Чувствительность к выбору clip epsilon.

## 5. Обсуждение

### 5.1 Что работает

1. **Формирующая награда (reward shaping)** — прогресс-компонента с масштабом 5.0 даёт плотный сигнал, без которого обучение на разреженной награде (+100 за цель) крайне медленное.
2. **Rollout буферизация** (Actor-Critic, TRPO, PPO) — накопление 8 эпизодов перед обновлением снижает дисперсию градиента.
3. **GAE** — баланс bias/variance через lambda = 0.95 ускоряет сходимость.
4. **Энтропийная регуляризация** — предотвращает преждевременный коллапс политики.
5. **Gradient clipping** — предотвращает взрывные обновления.

### 5.2 Сравнение алгоритмов

**TRPO** показал лучший результат: 100% success rate и кратчайшие траектории (103.2 шага). Ограничение KL-дивергенции обеспечивает монотонное улучшение и предотвращает деградацию политики.

**REINFORCE + Baseline** — второй по качеству (100% success). Baseline существенно снижает дисперсию по сравнению с ванильным REINFORCE.

**Actor-Critic** — 90% success, промежуточный результат. TD-бутстрэпинг вносит смещение, но более частые обновления компенсируют это.

**REINFORCE** — 85% success. Высокая дисперсия Monte-Carlo возвратов затрудняет стабильное обучение.

**PPO** — 70% success. Меньшая производительность связана с чувствительностью к гиперпараметрам (clip epsilon, число эпох) в данной задаче.

### 5.3 Что можно улучшить

1. **Подбор гиперпараметров PPO** — grid search или Optuna для clip_epsilon, ppo_epochs, mini_batch_size.
2. **Нормализация наблюдений** (running mean/std) — может ускорить сходимость.
3. **Curriculum learning** — постепенное усложнение (увеличение расстояния, уменьшение goal_radius).
4. **Larger networks** — более глубокие или широкие MLP для сложных стратегий обхода.
5. **Prioritized experience** — для off-policy расширений.

## 6. Архитектура нейросетей

### Policy Network (Gaussian)

```
Input (8) -> Linear(128) -> Tanh -> Linear(128) -> Tanh -> Linear(2) [mean]
                                                          + log_std (learnable, clamped [-2, 0.5])
```

Действие: a ~ N(mu(s), diag(exp(log_std)^2))

### Value Network

```
Input (8) -> Linear(128) -> Tanh -> Linear(128) -> Tanh -> Linear(1)
```

Выход: скаляр V(s).
